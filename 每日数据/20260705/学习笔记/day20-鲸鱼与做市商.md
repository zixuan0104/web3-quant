# Day 20 学习笔记 — 鲸鱼追踪 + 订单簿 + 交易所上币

## 一、链上数据的三个层次

### 1.1 第一层：鲸鱼转账方向分类

不是所有大额转账都是信号。关键看**方向**：

| 转账方向 | 含义 | 信号 |
|---------|------|------|
| 未知钱包 → 交易所 | 币转到交易所 = 准备卖 | 🔴 潜在卖压 |
| 交易所 → 未知钱包 | 提币到冷钱包 = 长期持有 | 🟢 潜在积累 |
| 交易所 → 交易所 | 做市商调仓/交易所内部调仓 | ⚪ 忽略 |
| 做市商 → 交易所 | 做市商准备在新的交易所做市 | 🟡 关注 |

**如果只盯着"大额转账"看，不做方向分类 = 把噪声当信号。**

### 1.2 第二层：聪明钱追踪

聪明钱地址追踪的价值不在单个地址，在**多地址同向操作**：

```
1 个聪明钱买 SOL → 可能只是个人偏好
3 个聪明钱买 SOL → 值得关注
5+ 个聪明钱同时买 SOL → 高置信度信号
```

我们的默认地址库只有 5 个聪明钱地址。实际运行中应该扩到 20-50 个，并持续追踪胜率。

### 1.3 第三层：交易所余额趋势

```
交易所 BTC 余额持续下降 → 积累 → 看涨
交易所 BTC 余额持续上升 → 抛压 → 看跌
稳定币交易所余额上升 → 增量资金准备入场
```

这是宏观指标，不触发具体操作，但影响整体仓位。

## 二、订单簿的四种做市商信号

### 2.1 假挂单（Spoofing）

```
特征：
  - 某个价位出现巨量挂单（> 平均深度 5 倍）
  - 存活 < 5 分钟
  - 撤单后不再补

含义：做市商在误导 — 制造虚假的支撑/压力感
```

检测假挂单的价值：如果一个币频繁出现假单 → 做市商不老实 → 降权。

### 2.2 冰山订单

```
特征：
  - 同一价位反复出现相同大小的挂单
  - > 10 次/小时
  - 被吃掉后自动补上

含义：
  - 买单冰山 = 有人在大规模吸筹，不想让人知道
  - 卖单冰山 = 有人在大规模出货
```

冰山订单是**加分项**——说明有大资金在认真运作，不是散户的随机行为。

### 2.3 深度不对称 — 关键：必须和价格行为一起看

```
买墙 + 价格不涨 = 假支撑（有人挂假买单出货）
卖墙 + 价格不跌 = 假压力（有人压价吸筹）
买墙 + 价格涨 = 真支撑（真实买盘）
卖墙 + 价格跌 = 真压力（真实卖盘）
```

**单独看深度比 = 没有信息量。深度比 × 价格行为 = 才有。**

### 2.4 做市商建仓/出货特征

```
建仓：窄幅横盘 + 买盘深度持续增加 > 6h + 卖盘不变
出货：窄幅横盘 + 卖盘深度持续增加 > 6h + 买盘不变
```

这是订单簿最重权重的信号（35%）。因为它反映的不是瞬时行为，而是持续一段时间的倾向。

## 三、交易所上币 — 四阶段模型

### 3.1 币安的四阶段

```
传闻期 → 不参与（信息不对称，你天然在劣势方）
公告期 → 记录时间戳和价格，但不追（HFT 的地盘）
上线首日 → 远离（噪声占主导）
沉淀期（3-14天）→ 你的机会区
```

**为什么只做沉淀期？**

上线首日的波动是投机资金 + HFT 机器人在博弈。你没有延迟优势，没有流动性优势，进去就是被收割。

沉淀期的逻辑完全不同：投机资金退出了，真实买盘和做市商行为浮现。这时候你用的是**分析能力**，不是拼速度。

### 3.2 沉淀期的入场条件（必须同时满足）

1. 价格在 7 日均线上方企稳
2. 订单簿深度持续 3 天增长（做市商在托底）
3. 成交量从上线峰值萎缩 70%+ 后企稳
4. 项目叙事仍然成立（不是一波流题材）

### 3.3 二线所上币的信号价值

| 交易所 | 信号强度 | 风险 | 权重 |
|--------|---------|------|------|
| MEXC | 弱 | 流水线产品多 | 5/15 |
| Bitget | 中 | 有审核 | 5/15 |
| Bybit | 较强 | 审核较严 | 3/15 |
| KuCoin | 较强 | 审核较严 | 2/15 |

**MEXC + Bitget 同时上同一个币 = 信号增强。只有 MEXC = 可能是流水线产品。**

### 3.4 做市商信任度

```
Jump Crypto: 0.95
Wintermute: 0.90
GSR: 0.85
DWF Labs: 0.65  ← 争议大，信任度打折
未知: 0.40
```

新币如果有一个高信任度做市商在操作 = 流动性不会突然枯竭。做市商质量直接决定你能不能在该币上正常交易。

## 四、订单簿数据的存储策略

```
实时: 最近 24h 全量快照（内存 + SQLite）
归档: 超过 7 天 → 存每小时均值 + 关键统计量（Parquet）
异常事件: 永久保存（复盘用）
长期趋势: 每月一个"深度画像"（bid/ask 曲线形状特征）
```

**为什么不全存？** 每分钟一次快照 × 20 档 × 24h = 28800 行/天/币种。全量存储 2 年数据是巨大的存储开销，而且大部分数据没有复用价值。

## 五、CT-DDD 视角：链上 + 订单簿的可控度分析

```
链上数据:
  可控度: 0.0 — 你完全无法控制鲸鱼的行为
  可观测度: 0.6 — 转账可见，但意图不可见（是调仓还是卖？）
  可辨识度: 0.4 — 需要结合标签 + 上下文判断

订单簿数据:
  可控度: 0.0 — 你无法控制挂单
  可观测度: 0.8 — 挂单量和价格清晰可见
  可辨识度: 0.5 — 需要存活时间等衍生指标才能区分真假单

交易所公告:
  可控度: 0.0
  可观测度: 1.0 — 公告是公开的
  可辨识度: 0.7 — 四阶段模型提供了清晰的分析框架
```

三者都处于灰箱/黑箱域 → 只能通过 harness（仓位调节 + 风控降权）来利用。
