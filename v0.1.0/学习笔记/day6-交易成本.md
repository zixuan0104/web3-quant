# Day 6 学习笔记 — 交易成本与订单类型

## 一、成本是量化的"沉默杀手"

### 1.1 一个典型的高频小利润策略

```
回测年化 50%，每天交易 5 次，日均利润 0.2%
扣完双向 0.2% 手续费 + 0.05% 滑点 = 每笔成本 0.25%
每天成本 = 5 × 0.25% = 1.25%
每天净利润 = 0.2% - 1.25% = -1.05%

年化从 +50% → -93%
```

**这就是为什么成本模型必须内嵌在回测里，不能事后算。** 高频策略回测年化 50%，扣完成本变成 -10% 是常态。

### 1.2 成本的四个来源

```
总成本 = 手续费 + 滑点 + 价差 + 市场冲击

手续费: 交易所收的（固定费率，可查）
滑点:   你的单子推动了价格（动态，取决于波动率和流动性）
价差:   市价单跨 bid-ask 的隐含成本（taker 有，maker 无）
市场冲击: 大单对订单簿的影响（量越大越贵）
```

## 二、四层成本模型

### 2.1 层 1: 手续费

| 订单类型 | 费率 | 说明 |
|---------|------|------|
| Maker（限价单） | 0.075% | 挂单吃返佣，BNB 抵扣后 |
| Taker（市价单） | 0.075% | 吃流动性，BNB 抵扣后 |

币安普通用户基准费率是 Maker 0.1% / Taker 0.1%。BNB 抵扣打 75 折 → 0.075%。

### 2.2 层 2: 滑点（从固定到动态）

**旧模型（Day 1-4）**:
```
滑点 = 固定 0.05% — 无论什么币、什么行情，永远 0.05%
```

**新模型（Day 6）**:
```
滑点 = (ATR / price) × 流动性乘数

BTC @ $65,000, ATR(1h) = $500 → 基础滑点 0.77% × 乘数 1.0 = 77 bps
SOL @ $146,   ATR(1h) = $3   → 基础滑点 2.05% × 乘数 2.5 = 513 bps
Meme @ $0.01, ATR(1h) = 极大   → 基础滑点 很大 × 乘数 20  = 💀
```

**为什么 ATR 适合做滑点估计**：
- ATR 高 = 波动大 = 做市商风险大 = 收紧深度 = 滑点增加
- ATR 已经内化了波动率信息，不需要额外计算

### 2.3 层 3: 价差成本

市价单隐含的 bid-ask spread 成本。限价单不需要付这笔成本（因为你挂在一边等人来吃）。

| 流动性等级 | 典型价差 | 代表币种 |
|-----------|---------|---------|
| 高 | 0.01% (1 bps) | BTC/ETH/SOL |
| 中 | 0.03% (3 bps) | DOT/LINK/UNI |
| 低 | 0.10% (10 bps) | 小币种 |
| 微 | 0.50% (50 bps) | Meme/土狗 |

### 2.4 层 4: 市场冲击

单笔交易金额 / 日成交量的比例越大，你的单子对市场的影响越大：

```
$500 交易 / $1B 日成交量 → 冲击 ~0 bps（忽略不计）
$5,000 交易 / $10M 日成交量 → 冲击 ~150 bps（显著）
$50,000 交易 / $10M 日成交量 → 冲击 ~1500 bps（灾难，滑点吃掉全部利润）
```

**这就是为什么小币种不能做大仓位。** 不是策略的问题，是流动性的硬约束。

## 三、实际数据对比

### 3.1 不同流动性等级的双向总成本

用 BTC 当前价 $63,406 做基准（不跨等级比较绝对金额，只看比例）：

| 流动性 | 双向总成本 | 隐含盈亏要求 |
|--------|:---:|:---:|
| BTC/ETH 主流 | 27 bps (0.27%) | 每笔赚 0.27% 才不亏 |
| 中型币 | 46 bps (0.46%) | 每笔赚 0.46% 才不亏 |
| 小币种 | 95 bps (0.95%) | 每笔赚 0.95% 才不亏 |
| Meme/土狗 | 315 bps (3.15%) | 每笔赚 3.15% 才不亏 |

**核心认知**：小币种的成本是大币种的 3-12 倍。一个策略在 BTC 上盈利不代表在 Meme 上也能盈利——可能成本就把利润吃光了。

### 3.2 Maker vs Taker（BTC 1h 实测 17,520 根 K 线）

| 指标 | 市价单 | 限价单 | 差异 |
|------|:---:|:---:|:---:|
| 交易笔数 | 268 | 259 | -9 |
| 成交率 | 100% | 98.3% | -1.7% |
| 总收益率 | -80.56% | -79.51% | **+1.04%** |
| 平均成本/笔 | 27 bps | 24.9 bps | -2.1 bps |

限价单比市价单多赚了 1.04 个百分点（在亏损策略上就是少亏 1.04%），代价是错失了 9 个信号。

### 3.3 成本吃掉了多少利润？

趋势跟踪策略在 2 年回测期内：
- 毛利润（不计成本）: -88.81%
- 总成本: 72.36%
- 净利润: -161.17%
- 24 笔交易因为成本从盈利变亏损（占比 9%）

**成本是最大的敌人，不只是对高频策略，对中频策略也一样。**

## 四、限价单 vs 市价单：什么时候用哪个

### 4.1 限价单的优势和代价

| 优势 | 代价 |
|------|------|
| Maker 费率更低（有时甚至负费率 = 返佣） | 可能不成交，错失行情 |
| 无价差成本（不跨 bid-ask） | 需要价格触达限价（多等几根 K 线） |
| 没有市场冲击（被动挂单） | 趋势市可能一路追不上 |

### 4.2 决策框架

```
选限价单的场景：
  ✅ 策略不依赖精确的入场时机（差几根 K 线无所谓）
  ✅ 交易的是流动性好的币种（BTC/ETH/SOL）
  ✅ 策略的 edge 比较小，成本是决定性因素
  ✅ 震荡市 — 价格会反复触及你的限价

选市价单的场景：
  ✅ 策略对时机敏感（突破策略 — 差一根 K 线就错过）
  ✅ 交易的是流动性差的币种（限价单可能永远不成交）
  ✅ 趋势市 — 价格快速远离你的限价
  ✅ 止损出场 — 不想因为限价不成交而扩大亏损
```

## 五、成本模型的 CT-DDD 视角

### 5.1 成本模型属于哪个域？

成本模型本身是**白箱域**——它内部的公式、参数、计算流程都是确定的、可测试的、可审计的。

但成本模型的输入（市场波动率、价差、流动性）来自**灰箱域**（交易所订单簿）——你只能观测到部分数据，交易所的内部撮合引擎是你不知道的。

### 5.2 如何在回测中处理这个灰箱

1. **成本模型用乐观/中位/悲观三档估计**，不依赖单一假设
2. **模拟盘验证成本模型的准确性**——偏差分析告诉你假设是否合理
3. **悲观假设作为实盘风控参数**——宁可高估成本也不低估

## 六、与现有回测引擎的集成

### 6.1 BacktestCostAdapter

```python
adapter = BacktestCostAdapter(CostModel(OrderType.TAKER, LiquidityTier.HIGH))
# 在回测策略中替换旧成本
fee_cost = adapter.round_trip_fee(price, atr)     # 替代 FEE_RATE * 2
slippage = adapter.entry_slippage(price, atr)      # 替代 SLIPPAGE
```

不破坏现有回测引擎，通过适配器模式逐步升级。

### 6.2 渐进替换策略

1. 先用新 CostModel 跑模拟盘，对比旧回测
2. 确认偏差来源合理后，把 CostModel 注入回测引擎
3. 旧固定成本逻辑保留（兼容已有策略），新策略默认用 CostModel
