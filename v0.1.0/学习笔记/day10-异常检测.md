# Day 10 — 异常检测

> 2026-07-06 | Week 2 第三天

## 学了什么

### 币圈「黑天鹅」是日常

传统金融里的"极端事件"在币圈是日常操作：
- 5 分钟跌 30% → 每周至少一次
- API 突然返回 503 → 币安维护/限流/挂了
- WebSocket 断了但没通知你 → 你以为在交易，其实在看 3 分钟前的价格
- 成交量 10x 突增 → 有人在比你更快地行动

异常检测不是"万一"的保险，是"必然"的防线。

### 三大检测维度

| 维度 | 问题 | 检测方式 |
|------|------|---------|
| **价格异常** | 价格在正常波动范围内吗？ | 滑动窗口极值 + 百分比阈值 |
| **连接异常** | 我们还在和交易所对话吗？ | API 错误率 + 心跳超时 + 重连计数 |
| **数据异常** | 收到的数据可信吗？ | 时间戳单调性 + 双源交叉验证 + 数据间隔 |

### 为什么错误率比单次错误重要

单次 API 超时 → 重试就行。
连续 5 次超时 → 交易所可能挂了。

错误率窗口（最近 10 次请求中 >30% 失败）比单次错误更能区分"偶发网络抖动"和"交易所宕机"。

### API 重试的指数退避

```
第 1 次重试: 等 1 秒
第 2 次重试: 等 2 秒
第 3 次重试: 等 4 秒
→ 如果全部失败且错误率超过阈值 → 熔断
```

错误率检查在重试之上——不是因为重试失败就熔断，而是因为错误率过高说明不是偶发问题。

### 熔断 & 自动恢复

熔断不是永久的：
- 价格断崖 → 暂停 10 分钟（等市场稳定）
- API 熔断 → 暂停直到错误率恢复
- 到期自动解除，不需要人工干预

但熔断期间绝对不允许新开仓。

1. **`anomaly_detector.py`** — 异常检测系统（450+ 行）
