## 学了什么

### 凯利公式 (Kelly Criterion)

公式：`f* = (p × b - q) / b`

- `p` = 胜率
- `q` = 1 - p（败率）
- `b` = 盈亏比 (avg_win / avg_loss)

**核心结论**：即使 60% 胜率、2:1 盈亏比的策略，最优仓位也只有 **20%**。超过这个比例，长期一定会爆仓——不是概率问题，是数学问题。

### 关键直觉

```
一个策略的长期增长率 ≈ (平均每笔收益) - (方差 / 2)

仓位翻倍 → 收益翻倍 → 但方差翻四倍 → 净增长率反而下降
```

这就是为什么：
- 重仓 = 高方差 = 高爆仓概率
- 凯利公式找的是"最优增长率"的仓位，不是"最大收益"的仓位
- 一旦超过凯利仓位，追加的每一分钱都在**降低**长期增长率

### 半凯利 vs 完整凯利

| 变体 | 仓位 | 风险 | 适用场景 |
|------|------|------|---------|
| 完整凯利 | f* | 最高增长率，但波动极大 | 理论上最优，实际很少用 |
| **半凯利** | f* / 2 | 3/4 的增长率，1/4 的波动 | **推荐** — 实际参数总有误差 |
| 1/4 凯利 | f* / 4 | 更保守 | 参数高度不确定时 |

实际中 p 和 b 都是估计值，有误差。半凯利是对"参数不确定"的保险。

### 仓位管理方法对比

| 方法 | 逻辑 | 优点 | 缺点 |
|------|------|------|------|
| 固定分数 | 每笔 2% 本金 | 简单、不会爆 | 不管策略质量 |
| 凯利 | f* = edge / odds | 数学最优 | 需要准确的 p 和 b |
| ATR 调整 | 宽止损 → 小仓位 | 自适应波动 | 依赖 ATR 计算 |

### 单币种敞口的教训

币圈的集中风险远比传统市场大：
- 一个币一天跌 30% 是日常
- 不要"觉得看懂了一个币"就重仓
- 单币种敞口硬编码上限，不能靠自觉

### 盈利提现策略

`赚了钱后飘了，加大杠杆爆仓` → 这是最经典的归零路径。

解决方案：
- 定期（如每 30 天）将利润的一部分转出交易所
- 提现后本金变小 → 仓位自然变小 → 防止"赢家效应"导致的过度自信
- 钱出了交易所才是你的钱

## 核心代码片段

```python
# 凯利公式核心
def kelly_criterion(win_rate, avg_win_pct, avg_loss_pct):
    b = avg_win_pct / avg_loss_pct  # 盈亏比
    q = 1.0 - win_rate
    f_star = (win_rate * b - q) / b
    return max(0.0, min(f_star, 1.0))

# ATR 波动率调整
# 仓位 = 目标亏损金额 / (ATR × 止损倍数)
# 高 ATR → 宽止损 → 小仓位
size_pct = target_risk_pct / (multiplier * atr / price)
```

## 思维记录

- 仓位管理是唯一一个"数学上可以证明你做错了"的环节。其他决策都有主观判断成分，但凯利公式是数学定理——仓位超过 f*，长期期望增长率为负。
- 风控（RiskManager）和仓位管理（PositionSizer）的关系：前者是门禁（过不过），后者是量尺（多少量）。两者不能互相替代。
- 盈利提现不只是财务操作，更是心理防护——钱少了，自然更谨慎。
- 蒙特卡洛模拟很直观：即使"好策略"用满凯利仓位，200 笔交易后也有概率爆仓。半凯利能大幅降低这个概率。
