# Day 9 — 分层止损

## 学了什么

### 止损的四层含义

止损不是「亏了就跑」那么简单。每一层回答一个不同的问题：

| 层 | 回答的问题 | 触发条件 | 数学表达 |
|----|-----------|---------|---------|
| L1 技术止损 | 「价格跌破关键位置了吗？」 | price ≤ entry - ATR × 2 | 硬约束，亏损 ≤ 2% |
| L2 追踪止损 | 「市场波动变了吗？」 | 盈利后回撤超过 3 个 ATR | 锁利润 = 最高价 - ATR × 3 |
| L3 时间止损 | 「我等了够久它不动，还等吗？」 | 持仓 N 根 K 线盈利不足 | 机会成本 > 预期收益 |
| L4 策略止损 | 「当初开仓的理由还成立吗？」 | 策略核心假设被证伪 | ADX < 15 → 趋势策略不成立 |

### 关键洞察

**L4 > L3 > L2 > L1 的优先级**：
- 策略逻辑止损最优先（根本性原因——你当初为什么进场已经不成立了）
- 时间止损其次（资金的时间成本）
- 追踪止损再次（锁住利润）
- 技术止损最后（最后的底线）

**追踪止损的「只紧不松」原则**：
```
多头: stop_price = max(当前ATR止损, 上一bar的止损价)
空头: stop_price = min(当前ATR止损, 上一bar的止损价)
```
止损只能朝有利方向移动——一旦收紧就不能再放松。

**硬上限优先于 ATR 止损**：
```
ATR 止损 = entry ± 2400（可能是 3.7% 的亏损）
硬上限   = entry ± 2%（绝对不亏超过 2%）
→ 取更紧的那个
```
如果 ATR 特别大（高波动），硬上限会接管。这防止了「止损太远导致一笔亏太多」。

### 策略逻辑止损的个性化

不同策略的「存在理由」不同：

| 策略 | 存在理由 | 失效条件 |
|------|---------|---------|
| 趋势跟踪 | 市场有趋势 | ADX < 15（无趋势） |
| 动量 | 价格惯性 | ROC 衰减到接近零 |
| 突破 | 突破是真的 | 价格回落到通道中线以下 |
| 均值回归 | 偏离会回归 | 偏离继续扩大（新趋势形成） |
| 费率套利 | 有利可套 | 资金费率降到不值得 |

## 风控环节注意

- StopLossManager 计算止损价，RiskManager 有最终否决权
- `StopLossRiskAdapter` 封装这个流程：止损信号 → 风控门禁 → 决策
- 风控说不平就不平——没有任何代码路径绕过风控层
- 止损系统的配置参数（ATR 倍数、时间上限等）可由 .env 覆盖
