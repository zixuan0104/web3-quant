# Day 21 学习笔记 — 信号融合

## 一、多源信号融合的核心问题

### 1.1 为什么需要融合

单独看每个信号源都有严重缺陷：

| 信号源 | 单独看的问题 |
|--------|------------|
| 链上扫描 | 只能看到"发生了什么"，看不到"为什么" |
| KOL 情绪 | 滞后、可能被付费操纵 |
| 二线所上币 | MEXC 假信号多，单靠它不可靠 |
| 币安 Alpha | 到这一步涨幅主力已过 |
| 社区情绪 | 噪声大，日常波动没有信息量 |

**但五路信号叠加在一起，交叉验证后，信号的可靠性就大幅提高。**

### 1.2 融合的逻辑不是"加总"，是"交叉验证"

```
链上说"有人在大买" + KOL 说"这是下一个 ANSEM"
  → 可能是真趋势，也可能是 KOL 在帮大户出货

链上说"有人在大买" + KOL 沉默 + 社区没反应
  → 更可能是真趋势——大户在默默吸筹，还没开始宣传
```

所以融合评分不只是简单加总。**信号来源的独立性越高，交叉验证的价值越大。**

## 二、五路信号的权重设计逻辑

### 2.1 主流币权重

```
链上扫描: 0-40 分（最高 — BTC/ETH 的链上数据最可靠）
KOL 情绪: 0-25 分
二线所:   0-15 分（主流币早已上线，此信号通常为 0）
币安 Alpha: 0-20 分（同上）
社区情绪: 0-10 分
```

### 2.2 Meme 币权重（隐式不同）

Meme 币评估侧重不同：
- 链上权重更高 — 新代币，链上数据是唯一真实的数据
- KOL 权重中 — 早期发现靠 KOL
- 交易所权重低 — 还没上大所
- 订单簿不可用 — 没有深度数据

### 2.3 警报分级

```
> 70 分 → 🔴 强力信号 → Telegram 推送 + 可考虑入场
50-70 分 → 🟡 关注信号 → Dashboard 中优先级 + 继续观察
30-50 分 → 🟢 观察信号 → 仅记录
< 30 分 → ⚪ 噪音 → 忽略
```

## 三、信号分层用途 — 最核心的设计

### 3.1 只有策略信号能触发开仓

| 信号来源 | 控制仓位 | 解释涨跌 | 发现机会 | **触发开仓** |
|---------|---------|---------|---------|------------|
| 策略信号 | — | — | — | **✅** |
| KOL 情绪 | ✅ 降权 | ✅ | ✅ | ❌ |
| 链上鲸鱼 | ✅ 降权 | ✅ | ✅ | ❌ |
| 做市商行为 | ✅ 降权 | ✅ | ✅ | ❌ |
| 市场环境 | ✅ 调仓 | ✅ | — | ❌ |

### 3.2 为什么这个边界如此重要

如果让 KOL 信号触发开仓 → 你在跟单。
如果让鲸鱼信号触发开仓 → 你在猜大户意图。
如果让交易所上币信号触发开仓 → 你在炒消息。

**这些都不是你的 edge。** 你的 edge 是你的策略逻辑——白箱域里经过回测验证的东西。

外部信号在灰箱/黑箱域 → 只能形成 harness（控制壳），调节策略的执行参数，不能替代策略本身。

## 四、仓位调整机制

### 4.1 融合评分 → 仓位系数

```python
> 70 分 → 仓位 ×1.0（正常）
50-70 分 → 仓位 ×1.0（关注但不减仓）
30-50 分 → 仓位 ×0.7（轻度减仓）
< 30 分 → 仓位 ×0.3（大幅减仓，依赖策略自身信号）
有 🚨 标记 → 仓位 ×0.0（暂停）
有 ⚠️ 标记 → 仓位 ×0.5（减半）
```

### 4.2 Meme 币仓位更保守

```
🔴 信号 → 0.5% 仓位（单笔上限）
🟡 信号 → 0.2% 仓位
🟢 信号 → 不入场
有风险标记 → 不入场
```

主流币的仓位调整系数基于回测验证的策略逻辑。Meme 币的仓位基于非对称收益的数学（10 个中 1 个 100x = +18.2%）。

## 五、ANSEM 案例复盘（信号融合推演）

假设我们在 ANSEM 爆发前（创建 6h 时）运行融合引擎：

```
链上(DexScreener成交量>$1.2M+聪明钱建仓)    35分
KOL(Solana生态多位开发者讨论)               20分
二线所(MEXC已上线)                            3分
币安(未上)                                   0分
社区(Twitter讨论量暴增)                       5分
─────────────────────────────────────────────
综合: 63分 🟡 关注信号

建议：0.2% 仓位入场，翻倍出本金
```

如果我们在 63 分时 0.2% 仓位入场：
- 成本：0.2% 总资金
- 如果归零：-0.2%
- 如果 100x：+20%
- 净期望：假设 10% 成功率 → +1.82%

**这就是 10-100x 框架的数学——用小到可以忽略的成本去博非对称收益。**

## 六、三维复盘集成

### 6.1 事件驱动层的具体形式

每笔大盈亏交易，自动回溯当时的外部信号：

```
📌 事件: SOL +8.5% @ 2026-07-05 14:30
   叙事驱动: KOL 驱动
   链上贡献: 3 笔鲸鱼转账在事件附近
   KOL 贡献: 5 次 KOL 提及在事件附近
   交易所贡献: 无显著事件
   社区贡献: 提及量暴增 3.4x
   ⚠️ 同期异常: KOL 集中提及（5个/12h）
```

### 6.2 归因的价值

归因帮你回答三个问题：
1. 这波涨是我的策略抓到的，还是运气？（靠外部信号涨的 = 不是策略 edge）
2. 为什么策略在这波跌的时候没跑掉？（查外部信号有没有提前预警）
3. KOL 喊的是不是真趋势？（回溯验证——喊完 7 天后看走法）

## 七、与回测策略的关系

```
回测策略（白箱域）        外部信号（灰箱域）
     │                       │
     │ 信号触发入场           │ 调节仓位大小
     │ 硬止损/止盈            │ 发现异常时降权
     │                       │
     └─────────┬─────────────┘
               │
         最终执行
    （策略方向 + 外部信号仓位系数）
```

**外部信号不能否定策略信号。** 策略说做多、外部信号差 → 做多，但仓位打 7 折。策略说做空、外部信号好 → 做空，但仓位不打折。策略说不动、外部信号好 → 不动——只有策略信号能开仓。

## 九、庄家动向追踪与开单点位（Day 21 增强）

### 9.1 庄家姿态判断 — 五维加权

```
维度 1: 鲸鱼链上行为（权重 25%）
  净流出交易所 = 积累 = +0.25
  净流入交易所 = 抛压 = -0.25

维度 2: 订单簿深度变化（权重 25%）
  买盘深度持续增加 > 6h = 吸筹 = +0.25
  卖盘深度持续增加 > 6h = 出货 = -0.25
  冰山订单发现 = 确认信号 +0.05

维度 3: OI + 资金费率（权重 20%，合约专属）
  OI↓ + 价格不跌 = 空头撤退/吸筹 = +0.15
  OI↑ + 价格不涨 = 多头拥挤/出货 = -0.15
  资金费率 > 50% 年化 = 多头拥挤 → -0.05
  资金费率 < -20% 年化 = 空头拥挤 → +0.05

维度 4: 聪明钱地址（权重 20%）
  5+ 钱包同向买入 = +0.20（strong）
  3-4 钱包 = +0.10（moderate）

维度 5: KOL 异常（权重 10%）
  C级KOL扎堆 = 付费出货 = -0.10
  KOL集群(high severity) = -0.05
```

**综合判定**：

| 总分 | 庄家姿态 | 建议 |
|------|---------|------|
| 吸筹 > 0.50 | 🔥 激进吸筹 | 只做多不做空 |
| 吸筹 > 0.25 | 📈 吸筹中 | 做多优先，做空谨慎 |
| 出货 > 0.50 | 🔥 激进出货 | 只做空不做多 |
| 出货 > 0.25 | 📉 出货中 | 做空优先，做多谨慎 |
| 差值 < 0.25 | ➖ 中性 | 多空皆可，轻仓按策略信号 |

### 9.2 合约特有信号

**OI-价格 四象限**：

```
OI↑ + 价格↑ = long_building   — 多头建仓（趋势延续）
OI↑ + 价格↓ = short_building  — 空头建仓（趋势延续）
OI↓ + 价格↑ = short_covering  — 空头撤退/逼空（可能反转）
OI↓ + 价格↓ = long_liquidating — 多头撤退/多杀多（可能反转）
```

**资金费率极端值**：
- 年化 > 50%：多头拥挤，容易多杀多 → 偏空
- 年化 < -20%：空头拥挤，容易逼空 → 偏多

### 9.3 开单点位计算

**现货点位**：
- 支撑位来源：订单簿买墙 > 2 BTC/ETH/SOL 的价位 + 鲸鱼平均成本 + ATR 回撤
- 阻力位来源：订单簿卖墙 > 2 BTC/ETH/SOL 的价位 + ATR 反弹
- 止损：支撑/阻力外放 0.5 ATR
- 止盈 1：1.5 ATR
- 止盈 2：3.0 ATR

**合约点位**（叠加上下文）：
- 下方清算密集区 = 支撑（做多目标）
- 上方清算密集区 = 阻力（做空目标）
- 止损更紧：清算位外放 0.3 ATR（合约杠杆放大风险）

**庄家姿态修正仓位**：
- 激进吸筹 → 做多点位仓位 ×1.5，做空点位仓位 ×0.3
- 激进出货 → 反之

### 9.4 BTC 示例（模拟数据推演）

```
BTC @ $65,000 | 庄家中性（置信度 95%）

现货做多: $64,500 买墙区 (SL:$63,900, TP1:$66,300, RR:3.0)
现货做空: $65,600 卖墙区 (SL:$66,200, TP1:$63,800, RR:3.0)
合约做多: $63,200 清算密集区 ($8.5M 爆仓量)
合约做空: $67,200 清算密集区 ($10.7M 爆仓量)
```

### 9.5 SOL 示例（模拟数据推演）

```
SOL @ $146 | 庄家中性（置信度 90%）

信号分解:
  🟢 聪明钱买入 +0.10（4 个钱包买入）
  🔴 OI+价格背离 -0.15（OI+8.2% + 价格+1.5% = 多头拥挤）
  🔴 资金费率 +35% 年化 = 偏空

合约做多: $138 清算密集区 ($1.2M 爆仓量)
合约做空: $155 清算密集区 ($2.1M 爆仓量)
```
