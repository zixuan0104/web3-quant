# Day 19 学习笔记 — KOL 监控与情绪分析

## 一、KOL 信息的本质：先有仓位，后喊单

### 1.1 为什么不能跟单

KOL 的工作是建立影响力，不是帮你赚钱。信息流顺序：

```
内部人建仓 → KOL 发现/被付费 → 喊单 → 散户看到 → FOMO 接盘
   ↑ 真正的 alpha 在这里      ↑ 你在这里看到的是滞后信息
```

KOL 喊单时，喊的人已经建仓了。你在这个时间点入场 = 帮他们抬轿子。

### 1.2 KOL 信号的真正价值

| 用途 | 怎么用 | 不用来做什么 |
|------|--------|------------|
| 解释价格波动 | "这波涨是因为 5 个 KOL 同时喊了" | 不用来预测涨跌 |
| 风控预警 | "KOL 集中喊单 → 可能是出货前奏 → 减仓" | 不用来开新仓 |
| 情绪极端标记 | "90% 的提及都是看多 → 情绪过热的概率很大" | 不用来跟单 |

**KOL 提及 ≠ 交易信号。** 这是 Day 19 最重要的认知。

## 二、KOL 分级体系

### 2.1 四级分类逻辑

| 级别 | 类型 | 信号价值 | 为什么 |
|------|------|----------|--------|
| **S 级** | 生态核心开发者 | 极高 | 他们关注的是技术趋势，不是价格。发现新项目的窗口最早 |
| **A 级** | 链上数据博主 / 高胜率交易员 | 高 | 有数据支撑，不是纯喊单。叠加链上信号可跟 |
| **B 级** | Alpha caller / 交易型 KOL | 中 | 可能比市场早一点发现趋势，但也可能是付费喊单 |
| **C 级** | 平时不提币，突然喊小币 | 低（大概率接盘） | 付费喊单的主要执行者 |

### 2.2 S/A 级的独特价值

S 级不是用来"跟单"的——你不会看到一个 Solana 核心开发者喊 "buy $X now"。他们的价值在于：
- 讨论的技术方向 = 6 个月后的热门赛道
- 关注的新项目 = 可能是下一个 ANSEM
- 对已有项目的评价 = 质量过滤

### 2.3 C 级 KOL 的反向参考价值

C 级 KOL 扎堆喊同一个币 → 大概率是付费喊单群在同步动作。这个信号的可靠度反而很高——**反向参考。**

## 三、五种社交异常检测

### 3.1 提及量暴增

```
24h 提及量 > 7 日均值 × 3 → medium severity
> 10x → high severity
```

但需要注意：提及暴增可能是因为真有利好（币安上币公告），也可能是因为 KOL 同步出货。**需要结合价格行为判断。**

### 3.2 KOL 集中提及

```
≥5 个 KOL 在 12h 内提及同一币种 → high severity
S/A 级 1 个顶 3 个
```

这是我们检测到 SOL 的异常——5 个 KOL 在 12h 内提及。说明 SOL 的叙事在加速传播。

### 3.3 社交热度 vs 价格背离

```
热度暴增 5x + 价格没动 → 有人在引导舆论
```

这是最有价值的异常信号。大户在吸筹时不会拉盘——他们会先铺舆论，价格不动。等散户涌入了才开始拉。

### 3.4 情绪极端逆转

```
24h 内从 80% 看多 → 80% 看空 → 黑天鹅
```

### 3.5 C 级 KOL 扎堆

```
3+ C 级 KOL 同时喊单 → critical severity
```

这是最危险的信号。C 级 KOL 通常是付费喊单的执行者，他们扎堆 = 大户在找出口。

## 四、DeepSeek API 选型逻辑

### 4.1 为什么不用 Claude 做情绪分析

| | Claude | DeepSeek |
|------|--------|----------|
| 成本 | ¥15-30/百万 tokens | ¥1/百万 tokens |
| 容量 | 适合深度分析 | 适合大批量简单分类 |
| 中文能力 | 好 | 极好 |

情绪分类是「量大但简单」的任务——每天可能有几十条推文需要分类。用 DeepSeek 省钱。

### 4.2 降级设计

API 不可用时，系统返回保守中性结果：
- `sentiment: "neutral"`, `confidence: 0.3`
- 不阻塞主流程

外部信号模块不能因为一个 API 挂了就全线瘫痪。

## 五、为什么 KOL 信号不能触发开仓（CT-DDD 视角）

KOL 信号属于**灰箱域**——可观测（你能看到谁在喊什么），但可控度极低（你无法控制 KOL 喊什么、无法验证喊单动机）。

```
KOL 信号的可控度: 0.1 — 你唯一能控制的是看不看
KOL 信号的可观测度: 0.7 — 你能看到提及频率和情绪，但看不到 KOL 的真实仓位
KOL 信号的可辨识度: 0.3 — 很难区分真信号和付费喊单
```

(0.1, 0.7, 0.3) → 灰箱域 → 只调节仓位，不触发开仓。**策略信号在白箱域（可控+可观测+可辨识），只有它能开仓。**

## 六、本日复盘

| 学到了什么 | 怎么用 |
|-----------|--------|
| KOL 喊单是滞后信号——喊的人已经建仓了 | 反过来用：KOL 集中喊单 = 减仓信号 |
| KOL 分级（S/A/B/C）决定信号权重 | S/A 级看趋势方向，B 级需链上验证，C 级反向参考 |
| 五种社交异常各有不同的严重级别 | C 级扎堆 > KOL 集中 > 情绪翻转 > 提及暴增 > 社交背离 |
| DeepSeek 做便宜的情绪分类，Claude 做深度分析 | 模型分工：贵的做重活，便宜的做轻活 |
| 外部信号是灰箱域 → harness 控制壳 | 调节仓位和归因，不触发开仓 |
| API 挂了不阻塞主流程 | 降级返回保守中性值 |
