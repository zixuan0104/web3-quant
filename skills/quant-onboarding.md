# 量化系统搭建引导

当用户表达"想做加密货币量化"但尚不清楚具体路线时，你必须按此流程引导。这是最高优先级的量化 skill——其他 11 个 skill 在路线确定后才调用。

## 0. 核心原则

1. **一次只问一个问题**：不要一次抛出一堆选项。逐个问，逐个答。
2. **用户说了算，但你给框架**：用户决定方向，你提供结构化的选项和 trade-off。
3. **直接客观的交流方式**：不画饼、不恐吓、不说"建议"当没说。每个建议必须有理由。
4. **实践优先，理论跟上**：先让用户跑出东西，再用结果反推学理论。反过来容易放弃。
5. **记住用户偏好**：用户明确说的偏好（如"我喜欢钻研逻辑"、"严格按策略执行"），写入 memory。

## 1. 需求澄清阶段（Phase 1: Discover）

按顺序逐个问，不要跳。每个问题根据前一个回答调整选项。

### Q1: 当前背景
```
你现在对编程和交易分别了解多少？
A. 有编程背景，金融新手
B. 有金融背景，编程新手
C. 两边都有基础，想深入
D. 两边都是新手
```

### Q2: 编程深度（如 Q1 选了编程背景）
```
你的编程能力大概在什么水平？
A. 能用 Python 写脚本、pandas 处理数据
B. 能独立开发中等应用，熟悉数据库/API
C. 有 C++/Rust/系统优化经验
D. 其他
```

### Q3: 交易经验
```
你在交易方面的基础？
A. 了解基本买卖和 K 线
B. 有手动交易经验，用过交易所 API，跑过简单自动化
C. 系统学过金融知识但缺实操
D. 其他
```

### Q4: 目标
```
你做量化的目标是什么？
A. 副业/兴趣，赚点额外收入
B. 希望做成主要收入来源，愿意投入大量时间
C. 职业化（进量化公司或自己做专业系统）
D. 先探索看看
```

### Q5: 时间投入
```
每天能投入多少时间？
A. 1-2h（业余）
B. 3-4h（比较充裕）
C. 5h+（几乎全职）
D. 时间不稳定
```

### Q6: 设备与环境
```
你现在用什么设备？在国内吗？
- 确认操作系统、网络环境（国内/海外）
- 确认是否已有 VPN / 云服务器
- 确认是否已有交易所账号
```

## 2. 路线设计阶段（Phase 2: Design）

### 2.1 提出 3 条路线

```
路线 A: 学术派（理论→实践）
路线 B: 实战派（以战养战）
路线 C: 螺旋上升（实践+理论交替 — 推荐）
```

给出每条路线的优劣和推荐理由。绝大多数情况推荐 C。

### 2.2 确定时间框架

根据用户的时间投入，给出两种节奏：
- **冲刺月**（适合每天 3-4h）：4 周可交付成果
- **慢速版**（适合每天 1-2h）：8 周

### 2.3 设计逐日/逐周计划

必须包含：
- 每天的具体主题
- 学什么（理论）
- 做什么（实践）
- 产出什么（可交付物）
- 用什么 skill（从 skills/ 目录选）

### 2.4 确认策略类型偏好

在路线中安排"多策略试跑"阶段（Day 3-4），让用户通过实战感受选择：

| 你能接受什么痛苦 | 对应策略 |
|------|------|
| 连续亏损 6-7 次 | 趋势跟踪 |
| 小赚多次但讨厌大亏 | 均值回归 |
| 喜欢数据和统计 | 多因子/动量 |
| 不确定 | 三种都跑 |

## 3. 技术栈确认阶段（Phase 3: Setup）

### 3.1 数据源
```
必选：币安 + OKX（双源交叉验证）
可选：CoinGecko（基准价）、合约 OI、链上数据

Layer 1: 现货 K 线（Day 1）
Layer 2: 合约 OI + 资金费率（Week 2）
Layer 3: BTC.D + 恐惧贪婪（Week 2）
Layer 4: 链上 + KOL（Week 3-4）
```

### 3.2 回测框架
```
推荐：backtrader（事件驱动）+ vectorbt（向量化扫描）
备选：Freqtrade（开箱即用的完整框架）
不要：自己写回测引擎
```

### 3.3 云服务器
```
推荐：Vultr 东京 $6/月 或 AWS Lightsail 东京 $5/月
用途：策略运行环境，延迟 <10ms vs 国内直连 200ms+
注意：VPN 挂代理 ≠ 云服务器。前者是本地跑+伪装位置，后者是代码跑在云端。
```

### 3.4 AI 模型
```
主力：Claude Sonnet（API，策略设计+报告生成）
轻量：DeepSeek（KOL 情绪分析，¥1/百万 tokens）
月费：¥100-300（冲刺月），¥100-200（稳定运行后）
```

### 3.5 提醒推送
```
首选：Telegram Bot（币圈标配，免费，支持交互）
备胎：Bark/PushDeer（国内直连，纯通知）
高级：扣子（AI 对话分析，第二个月再加）
```

## 4. 环境踩坑阶段（Phase 4: Debug）

实际执行中必然遇到以下坑，每个都给出解决方案：

### 4.1 Windows GBK 编码
```
症状：print 中文/emoji 报 UnicodeEncodeError
修复：脚本顶部添加
  import sys
  if sys.platform == 'win32':
      sys.stdout.reconfigure(encoding='utf-8', errors='replace')
```

### 4.2 国内 API 连接超时
```
症状：ccxt 连接 binance.com 超时
诊断：python -c "import ccxt; ccxt.binance().fetch_ticker('BTC/USDT')"
修复：
  - 方案 A：ccxt 配置 proxies 参数走 VPN 代理
  - 方案 B：代码部署到东京云服务器
  - 注意：Clash 默认代理 127.0.0.1:7890
```

### 4.3 数据时间戳
```
规则：北京时间（UTC+8）为日期界限
代码：datetime.now(pytz.timezone('Asia/Shanghai'))
文件夹：YYYYMMDD 格式，如 20260705
```

### 4.4 API 限频
```
ccxt 配置：'enableRateLimit': True
自定义：exchange.rateLimit = 1200（币安默认 1200ms）
重试：3 次，指数退避
```

## 关键话术

以下措辞经用户确认有效，可复用：

- "这个策略赚钱的原因是：_________"（强制用户写赚钱逻辑）
- "你能接受哪种死法？"（帮用户选策略类型）
- "回测的目的是证伪策略，不是证明策略"
- "慢就是快"
- "当你解释不了自己的策略为什么赚钱的时候，它就已经失效了"
- "不是所有回撤都是策略失效"（市场周期切换 vs 策略逻辑错误）
- "VPN 挂代理 ≠ 云服务器"
