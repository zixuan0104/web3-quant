# 风控检查清单

策略从回测到实盘，必须通过这份检查清单。任何一项标红 → 不能上线。

## 硬性规则

1. **风控否决权**：风控层的代码优先级高于策略层。策略说"买"，风控说"不能买"，结果是不买。
2. **分层止损必配**：单笔止损 + 日内熔断 + 策略止损 + 时间止损，缺一不可。
3. **仓位硬上限**：所有仓位上限硬编码，修改必须人工确认。
4. **最小权限原则**：API key 不开提现、不开转账，交易权限仅限现货（初期）。

## 上线前检查清单

### 🔴 致命级（不通过 = 不能上线）

| # | 检查项 | 怎么验证 |
|------|------|------|
| 1 | API key 没有提现权限 | 去交易所 API 管理后台确认，截图留存 |
| 2 | API key 有 IP 白名单限制 | 绑定你的东京服务器 IP，不允许其他 IP 调用 |
| 3 | 单笔最大亏损上限已设（本金的 1-2%） | 代码里硬编码，不是配置文件 |
| 4 | 日内亏损熔断已设（本金的 5%） | 代码里累计当日盈亏，达标自动停止新开仓 |
| 5 | 止损单逻辑已验证（模拟盘至少触发过 1 次） | 看模拟盘日志，确认止损单能被触发且正确执行 |
| 6 | 策略不能同时持有相反方向仓位 | 代码里检查：已有买单时禁止开卖单，反之亦然 |
| 7 | 服务器时区与交易所一致（UTC） | `date` 命令确认服务器时间，偏差 >5 秒报警 |

### 🟠 重要级（不通过 = 减仓 50% 上线）

| # | 检查项 | 怎么验证 |
|------|------|------|
| 8 | 成本模型精度验证通过（实盘滑点 vs 回测假设偏差 < 0.1%） | 模拟盘 10 笔以上成交，对比回测假设 |
| 9 | 资金费率计入持仓成本（如做合约） | 回测里验证资金费率模型 |
| 10 | 极端行情保护：价格 5 分钟内波动 >10% 自动暂停 | 代码里监控，模拟触发一次 |
| 11 | WebSocket 断线重连后状态恢复正确 | 手动断网 30 秒后检查策略状态 |
| 12 | 夜盘/周末流动性低的时段自动降仓位 | 按时间段自动调整仓位上限 |

### 🟡 建议级（不通过 = 标记注意，可上线）

| # | 检查项 |
|------|------|
| 13 | 交易所费率变动监控（每周拉取最新费率） |
| 14 | 多交易所备份：A 交易所 API 异常时切换 B 交易所 |
| 15 | 盈利自动提现策略已制定（每赚 N U 提走 M%） |
| 16 | 税务记录完整（每笔交易含时间戳、价格、费用） |

## 风控模块代码模板

生成风控模块时必须包含以下结构和接口：

```python
class RiskManager:
    """
    风控管理器 — 策略的所有交易决策必须先经过本模块
    
    调用顺序：
    策略.generate_signal() → RiskManager.check() → 通过 → 执行
                                                → 拒绝 → 记录原因
    """
    def __init__(self, config):
        self.daily_pnl = 0.0           # 当日累计盈亏
        self.daily_loss_limit = config['daily_loss_limit']  # 日熔断线
        self.position_limit = config['position_limit']      # 单币仓位上限
        self.max_slippage = config['max_slippage']          # 最大允许滑点
        self.is_halted = False         # 熔断状态
        self.halt_reason = None
    
    def check_entry(self, signal):
        """
        开仓前检查，返回 (approved: bool, reason: str)
        
        检查顺序：
        1. 熔断状态？ → 拒绝
        2. 日内亏损是否超限？ → 拒绝
        3. 新仓位 + 现有仓位 > 上限？ → 拒绝
        4. 是否有同币种反向持仓？ → 拒绝（先平后开）
        5. 预期滑点是否在允许范围内？ → 拒绝
        6. 全部通过 → 批准，返回调整后的仓位大小
        """
        pass
    
    def check_exit(self, signal):
        """平仓检查（比开仓宽松，平仓不应该被阻断）"""
        pass
    
    def update_daily_pnl(self, pnl_change):
        """更新当日累计盈亏，检查是否触发熔断"""
        self.daily_pnl += pnl_change
        if self.daily_pnl <= -self.daily_loss_limit:
            self.is_halted = True
            self.halt_reason = f"日亏损熔断：{self.daily_pnl} <= {self.daily_loss_limit}"
            self.send_alert(self.halt_reason)  # Telegram 告警
    
    def check_market_extreme(self, price_data):
        """极端行情检测：最近 5 分钟波动 > 10% → 暂停"""
        recent_volatility = self.calc_volatility(price_data, window=5)
        if recent_volatility > 0.10:
            self.is_halted = True
            self.halt_reason = f"极端波动熔断：5分钟波幅 {recent_volatility:.1%}"
            self.send_alert(self.halt_reason)
    
    def reset_daily(self):
        """每日 UTC 00:00 重置当日统计"""
        self.daily_pnl = 0.0
        if not self.is_halted:
            pass  # 熔断状态需要人工解除
```

## 止损分层设计

```
Layer 1: 单笔止损（硬止损）
├── 每笔交易固定止损：本金的 1-2%
├── 由交易所止损单 + 程序双重保障
└── 触发后：平仓 + 记录 + 等待下一个信号

Layer 2: 日内熔断
├── 当日累计亏损达本金 5% → 停止新开仓
├── 已有仓位只允许平仓，不允许加仓
└── 次日 UTC 00:00 自动解除（或人工提前解除）

Layer 3: 策略止损（逻辑止损）
├── 策略的核心假设被证伪时退出
├── 例：趋势策略，趋势线破了就走
├── 例：均值回归，回归窗口超时不回归就走
└── 不等价格止损，逻辑错了就认

Layer 4: 时间止损
├── 持仓超过 N 小时没按预期走 → 平仓
└── "我赌它4小时内涨，4小时没涨就走了"
```

## 熔断后的恢复流程

```
1. 熔断触发 → Telegram 立即告警
2. 系统自动：禁止新开仓，已有仓位可平
3. 用户收到告警 → 人工判断：
   a. 是市场异常（极端行情）→ 等行情稳定后手动解除
   b. 是策略异常（连续亏损超出历史范围）→ 暂停策略，复盘
   c. 是代码 bug → 修 bug 后再上线
4. 解除熔断：用户发送指令 `/unhalt`（需二次确认）
```

## 错误案例（必须对照检查）

- ❌ 2019 年某量化团队：忘记设置单币种仓位上限，在 LUNA 上 overexposed，归零
- ❌ 2022 年 FTX 爆雷：交易所账户里的资产不是你的资产。盈利及时提现。
- ❌ 止损设了但用的是市价止损单：大行情下滑点巨大，止损在 -30% 才成交
- ❌ API key 开了提现权限：被钓鱼后账号清空
