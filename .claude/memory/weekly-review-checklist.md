# 周度复盘自问 Checklist

> 来源：Eugene Ng (0xENAS) — "如果我今天清仓，会以同样比例买回来吗？"
> 内化方式：不是自动化代码，是人每周手动执行的思考框架

## 触发条件

- **频率**：每周一次（建议周日晚上）
- **执行人**：你自己（LLM 辅助分析，但不替代判断）
- **Day 15-18 落地**：作为复盘报告模板的一部分，程序自动生成数据，人来回答这些问题

## 8 个核心问题

### 1. 「如果我今天清空所有持仓，会以同样比例买回来吗？」
- Eugene 的原话：答案通常是"不会"
- 如果不会 → 你的仓位配置有问题 → 该调仓
- 这是最重要的问题，每周必须诚实回答

### 2. 「本周赚钱的交易，是靠 Alpha 还是靠 Beta？」
- Alpha = 你的策略选币/择时能力
- Beta = 市场整体涨了，买什么都赚
- 如果大部分是 Beta → 策略没有超额收益 → 持有 BTC 就够了

### 3. 「本周亏钱的交易，是止损执行了还是抱着侥幸？」
- 止损执行了 → 系统在工作，亏损是策略成本
- 抱着侥幸（设了止损但撤了/放宽了）→ **这是最危险的行为**
- 统计：止损执行率应该 = 100%

### 4. 「有没有哪笔交易，我下注的理由和当初回测的假设不一样？」
- 回测假设 ≠ 实盘理由 → 策略漂移 (strategy drift)
- 策略漂移是所有量化策略慢慢失效的第一原因
- 如果有 → 记录偏差，下周盯紧

### 5. 「当前市场环境，我的 4 条策略各该分配多少仓位？」
- 看 `regime_classifier.dashboard()` 的策略乘数
- 实际仓位是否和理论仓位一致？
- 如果不一致 → 是主动 override 了还是风控拦了？记录原因

### 6. 「本周的外部信号有没有有效预警？」
- 统计：外部信号触发了多少次？对应行情后来怎么走的？
- 信号准确率 = 预警后真的跌了/涨了的比例
- 这是 auto_trigger 评估门禁的核心数据

### 7. 「有没有哪个持仓我'爱上'了？」
- Eugene：「不要和仓位谈恋爱」
- 客观判断标准：持仓时间 > 预期持有时间 2 倍 → 你爱上它了
- 这时候必须强制减仓或清仓

### 8. 「下周如果市场暴跌 30%，我的最大亏损是多少？」
- 用当前仓位 × 历史最大波动率估算
- 这个数字必须 < 你能承受的范围
- 如果超过 → 本周必须降仓

## 程序辅助（Day 15-18 实现）

复盘脚本 `weekly_review.py` 将自动生成以下数据，供人工回答上述问题：

```
weekly_review.py 输出:
├── 本周所有交易明细（含盈亏、持有时间、信号来源）
├── Alpha/Beta 分离估算
├── 止损执行率统计
├── 策略乘数 vs 实际仓位对比
├── 外部信号触发记录 + 准确率滚动统计
├── 持仓时间分析（标记"爱上"嫌疑的仓位）
└── 压力测试：暴跌 30% 的亏损估算
```
