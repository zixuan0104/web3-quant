# 28 天加密量化冲刺路线

> 目标：一个月内从零搭建完整的量化交易系统，小资金实盘运行，产出可迭代的策略和自动化报告。
> 
> 投入：每天 3-4 小时，总计 100-120 小时。
>
> 配套 Skill：`skills/` 目录下 11 个量化专用 skill，每个阶段标注了对应使用的 skill。

---

## 冲刺前准备（Day 0）

在 Day 1 之前搞定的基础设施：

- [ ] 东京云服务器（Vultr $6/月 或 AWS Lightsail $5/月），Ubuntu 22.04，Python 环境
- [ ] Anthropic Console 注册 + ZA Bank 卡绑上 + API key 到手
- [ ] Claude Code 配置 Anthropic API key
- [ ] DeepSeek API key（轻量任务用，¥1/百万 tokens）
- [ ] Telegram Bot 创建（`@BotFather` → 拿到 token）
- [ ] 币安交易所 API key（只开现货交易权限，不开提现，IP 白名单绑定服务器）
- [ ] OKX 交易所 API key（同上，交叉验证 + 备用）
- [ ] Git 仓库（GitHub 私有仓库，代码 + 配置分离，config 不入库）

### Day 4 — 均值回归 + 动量策略 + 资金费率套利

| 环节 | 内容 |
|------|------|
| **学** | 均值回归的逻辑（统计偏离度、布林带、RSI）、动量因子、资金费率套利原理 |
| **做** | 实现均值回归和动量策略。加入资金费率套利作为基准策略——其他策略必须先跑赢它。三策略对比报告 |
| **产出** | 三策略对比报告，选出最优进入模拟盘 |

**资金费率套利（最稳起步策略）**：
```
现货买入 + 合约做空 = 锁定价差，收取资金费率
- 风险最低，逻辑简单，可机械化执行
- 年化 10-30%，取决于市场条件
- 作为你的基准策略：其他策略必须跑赢它才有存在的意义
```

**策略类型决策参考**：

| 你能接受什么痛苦 | 对应策略 |
|------|------|
| 连续亏 6-7 次很正常 → 靠几笔大赚覆盖 | 趋势跟踪 |
| 小赚多次，但一次不止损的大亏会致命 | 均值回归 |
| 喜欢数据和统计规律 | 多因子/动量 |
| 不确定 → 三种都跑一遍，用数据选 | — |

**使用 Skill**：[quant-strategy](skills/quant-strategy.md) + [quant-backtest](skills/quant-backtest.md)

### Day 8 — 仓位管理

| 环节 | 内容 |
|------|------|
| **学** | 凯利公式（为什么重仓必爆）、固定分数 vs 波动率调整仓位、盈利提现策略 |
| **做** | 给实盘加入动态仓位计算模块。单币种最大敞口硬编码限制 |
| **产出** | 仓位管理模块 |

**为什么重仓必爆**：
```
凯利公式告诉你：即使你有一个 60% 胜率、盈亏比 2:1 的策略，
最优仓位也只有 20%。超过这个比例，长期一定会爆仓。
不是概率问题，是数学问题。
```

**防坑重点**（案例教训）：
- ❌ 赚了钱后飘了，加大杠杆爆仓 → ✅ 仓位上限硬编码 + 盈利自动提现
- ❌ 觉得自己看懂了一个币，重仓被割 → ✅ 单币种最大敞口限制

**策略健康度六维评分**（每日自动）：

| 指标 | 权重 |
|------|------|

**灯号动作**：

**参数滚动优化机制**：
```
每周自动执行：样本内(60天)优化 → 样本外(60天)验证
→ 验证通过（样本外 ≥ 样本内的 70%）→ 更新参数
→ 验证失败 → 保持旧参数，过拟合嫌疑
```

### Day 19 — KOL / 推特监控

| 环节 | 内容 |
|------|------|
| **学** | 非结构化数据引入量化的边界（辅助因子，不直接交易）、KOL 喊单的时间线分析 |

**关键原则**：
```
KOL 提及 ≠ 交易信号。

这个信号的真正价值是解释已有的价格波动（复盘归因用）

```

**防坑重点**：
- ❌ KOL 喊了就跟 → ✅ KOL 集中喊单通常不是好事——可能在同步出货
- ❌ 情绪分析替代策略信号 → ✅ 情绪是滞后指标

### Day 20 — 做市商 + 鲸鱼追踪

| 环节 | 内容 |
|------|------|
| **学** | 订单簿信号（冰山单、假挂单、深度不平衡）、链上数据（大额转账、聪明钱包、交易所余额） |
| **做** | 订单簿快照采集（每分钟）+ 链上数据每日拉取（Whale Alert / Arkham / Dune）→ 异常标记 |
| **产出** | 鲸鱼/做市商监控模块 |

**做市商行为信号**：

| 信号 | 含义 |
|------|------|
| 窄幅横盘 + 买盘深度持续增加 | 做市商在下面接货（建仓） |
| 窄幅横盘 + 卖盘深度持续增加 | 做市商在上面出货 |
| 固定价位巨量挂单反复出现 | 做市商在关键位布防 |
| 撤单后立刻在原点位补单 | 程序在维护，不是散户在挂 |

**链上数据用途**：
- 不作为开仓信号
- 用于：风控降权 + 复盘归因 + 机会发现

**使用 Skill**：[quant-onchain](skills/quant-onchain.md) + [quant-orderbook](skills/quant-orderbook.md)

## 整个系统的三层防御体系

```
层级 1: 日常防御（全自动）
├── 健康度仪表盘实时监控（quant-strategy-health）
├── 黄灯提醒 → 橙灯减仓 → 红灯暂停（quant-risk）
├── 市场环境分类器自动调整仓位（quant-market-regime）
└── Telegram 分级推送（Day 24）

层级 2: 周期防御（人工+程序）
├── 每周参数滚动优化（防过拟合验证）
├── 每月退役评估（quant-strategy-health）
└── 休眠策略唤醒检查

层级 3: 根基防御（人工）
├── 策略逻辑假设是否还成立？（月报因果复盘）
├── 市场结构有没有根本性变化？
├── 我还能解释这个策略为什么赚钱吗？
└── 底线：当你解释不了策略为什么赚钱，它就已经失效了
```
