# Day 2 学习笔记 — 回测框架

## 一、回测的底层逻辑

### 1.1 事件驱动的本质

回测不是「把历史数据喂给策略算个总收益」——那是纸上谈兵。真正的事件驱动回测逐根 K 线推演：

```
for 每根K线:
    策略.on_bar(K线) → 入场/出场/不动
    如果有交易 → 扣手续费 + 滑点 → 更新净值
```

每一根 K 线只能看到**那一刻之前**的数据。这是回测正确性的根基。

### 1.2 为什么不能自己写回测引擎

自己写的回测引擎最容易出三种 bug：

1. **未来函数**：信号生成用了「未来」的数据（比如用当日收盘价算当日信号 → 实盘中收盘价还没出来）
2. **订单管理缺失**：没有处理「订单未成交」「部分成交」「重复下单」的状态机
3. **净值计算错误**：浮动盈亏没按正确方向算，或者忘了扣手续费

backtrader / vectorbt 这些年被几十万人踩过的坑，比你从零写的要少得多。

### 1.3 撮合模型 = 成本模型

回测里必须内嵌成本，不能事后算。三个组成部分：

| 成本 | 典型值 | 说明 |
|------|--------|------|
| 手续费 | 0.1% 单边 | Maker/Taker 不同，币安用 BNB 扣费更低 |
| 滑点 | 0.05% 基础 + 流动性溢价 | 小币种滑点能吃掉全部利润 |
| 冲击成本 | 大单额外滑点 | 现货起步仓位小，暂时不建模 |

**关键认知**：一个高频小利润策略，回测年化 50%，扣完成本可能变成 -10%。成本模型必须内嵌在回测里。

## 二、过拟合：量化最大的敌人

### 2.1 什么是过拟合

> 你在回测里找到的参数组合，恰好拟合了历史数据的噪声，而不是真正的市场规律。

判断标准：**样本外表现 < 样本内的 50%** → 过拟合嫌疑。

### 2.2 为什么 ≤3 个参数

每多一个参数：
- 搜索空间指数增长（4 参数 × 5 取值 = 625 组合 → 5 参数 = 3125）
- 过拟合概率急剧上升
- 策略逻辑变得不可解释

**参数数量 = 自由度 = 过拟合风险**。3 个参数是经验最优上限。更多参数 → 你在拟合历史，不是在发现规律。

### 2.3 我们的实际数据

BTC 1d 趋势跟踪 EMA(10/40)：
- 样本内夏普：-0.50 → 样本外夏普：1.65
- 样本内亏钱、样本外赚钱 → **不是过拟合，是市场环境变了**

这种情况比「样本内高/样本外低」更健康——说明策略在真正的未来数据上有效，只是训练期正好是震荡市。

## 三、未来函数：最隐蔽的 bug

### 3.1 常见未来函数

| 错误 | 为什么是未来函数 | 正确做法 |
|------|-----------------|----------|
| `df['close']` 直接算信号 | 这根 K 线的收盘价在盘中不存在 | 用 `shift(1)` 取前一根 K 线的值 |
| `df['high'].rolling(20).max()` | rolling 默认包含当前值 | 同上，`.shift(1)` |
| 用全样本标准化（z-score） | 用到了未来的均值和标准差 | 用 expanding window |

### 3.2 我们的防护措施

`strategy_base.py` 里所有策略的 `precompute()` 都做了 `.shift(1)`：

```python
# ✅ 正确：用前一根 K 线的 EMA 值判断当前 K 线的信号
self._ema_fast = close.ewm(span=self.fast_period).mean().shift(1)
```

这意味着第 100 根 K 线做决策时，用的是第 99 根 K 线的 EMA 值——和实盘完全一致。

## 四、样本内/外分割的意义

### 4.1 为什么 70/30

- 70% 样本内：足够多的数据做参数优化
- 30% 样本外：足够验证策略是否真的有效
- 不按时间顺序随机分割 → **错误**（时序数据的自相关性被破坏了）

### 4.2 分割的真正目的

不是为了「证明策略好」——是为了「证伪策略」。如果样本外表现大幅低于样本内，策略大概率只是在拟合噪声。

**回测的目的是证伪策略，不是证明策略。**
